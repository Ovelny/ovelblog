Ethernet introduction

Ethernet is about connecting computers together in a local area network (LAN). It is used, among other things, by switches and routers.

Overview of an ethernet frame

From left to right, each part of an Ethernet frame goes like this:

  • Preamble: 7 bytes for synchronization
  • SFD: 1 byte for delimitation
  • Destination MAC: 6 bytes to carry the MAC address of the receiver
  • Source MAC: 6 bytes to carry the mac address of the sender
  • Type: 2 bytes to describe the type of data sent
  • Payload: between 46-1500 bytes of data sent
  • FCS: 4 bytes of CRC checksum to ensure data reception

The MAC address

  • MAC stands for Media Access Control address
  • It's a physical address, theoretically unique for each machine
  • Made of 6 bytes of data
  • Represented in hexadecimal notation
  • First half of the MAC address is the OUI (Organizational Unique Identifier), produced by the machine's manufacturer
  • Second half of the MAC address is the serial number, the unique part of the ID for each machine

Duplex

  • Half-duplex:
    • cannot receive and send data at the same time
    • hubs are half-duplex
  • Full-duplex:
    • receive and send data simultaneously
    • switch are full duplex

Half-duplex are rare nowadays. Full-duplex if the norm.

CSMA/CD

Protocol used to detect collisions with half-duplex. Not used anymore.

  • CS: Carrier Sense
    • check if a signal is already present or not to send data
  • MA: Multiple Access
    • permit to use more than one device on the network

  • CD: Collision Detect

    • check if data is colliding or not

  • Operation summary:

    • listen if the network is currently in use or not
    • if the network is free, send data
    • if a collision is detected:
    • send a JAM signal: tell all devices to stop transmitting for a while then begin to resend data

Full-duplex renders the CSMA/CD obsolete: since you can send and receive data simultaneously, there is no collisions and no need to check for them.

CSMA/CA: wireless networks

  • Use collision avoidance rather than collision detection
  • Use RTS / CTS: Ready To Send and Clean To Send signals
  • Solve the "hidden node" problem:
    • make a relay when machines cannot communicate directly with RTS and CTS
