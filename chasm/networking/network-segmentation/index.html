Network segmentation

  • Between each switch: no knowledge of the rest of the network
  • Broadcast: limited to switches
  • The default topology is to have many switches to segment the network
  • Or to use a more convenient alternative: VLANs
  • VLANs act as switches, but separates the network logically and not physically: it's more flexible and scalable
  • You can have multiple VLANs per switch, no limitations of segmentation
  • VLANs on multiple switches however, do not scale well
  • Trunking is a better alternative for this:
    • only one physical connection
    • can transmit multiple VLANs through that connection
  • 802.1Q trunking:
    • normal Ethernet frame, but with an additional VLAN header
    • the header is needed to determine the frame's destination
    • before 802.1Q, ISL was used but is now obsolete
