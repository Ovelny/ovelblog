Understanding the osi model

  • The OSI model stands for the Open Systems Interconnection reference model
  • It's a guide to understand and reason about how data is being processed, but is not meant to be an exact reference for everything
  • Not to be confused with the OSI protocol suite
  • Refer to it often to describe any issues with coworkers
  • Mnemonic way to remember all layers of the OSI model: All people seem to need data processing

Layers of the OSI model

  • Layer 1: physical

    • carrying the signal
    • cables, fiber, etc
    • the "physical" part of data processing

  • Layer 2: data link layer

    • the most essential form of networking
    • MAC addresses, DLC protocols, Ethernet

  • Layer 3: network layer

    • routers
    • IP protocol
    • crossing different networks
    • splitting data into smaller packets

  • Layer 4: transport layer

    • TCP and UDP
    • transfer data splitted into packets

  • Layer 5: session layer

    • managing communication between different machines
    • tunneling and control

  • Layer 6: presentation layer

    • data encryption, SSL and TLS
    • character encoding

  • Layer 7: application layer

    • the applications that users can see
    • closest layer from the end user
