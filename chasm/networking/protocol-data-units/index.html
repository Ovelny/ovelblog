Protocol data units

PDU overview

  • Unit of information sent to an OSI layer with a protocol like Ethernet, IP, TCP, etc
  • Protocols do not care about the content of data
  • Each PDU is added from top to bottom of the OSI model, layer by layer
  • Each layer adds a section to properly transmits data
  • When data is received, it goes bottom to top on the receiver end, and is stripped off off a section at each related layer

MTU

  • Stands for Maximum Transmission Unit
  • Maximum size of a packet before fragmentation
  • If a fragment is lost, it is retransmitted
  • The MTU might not be accurate when automatically configured
  • Usually configured once, and never changed afterwards

Example of an Ethernet frame

From left to right:

  • DLC header: 14 bytes
  • IP header: 20 bytes
  • TCP header: 20 bytes
  • TCP data: 1460 bytes
  • FCS: 4 bytes
  • Total IP packet size (IP header + TCP header + TCP data) = 1500 bytes

IP fragmentation: always fragmented in multiples of 8

DF: Don't Fragment bit

  • Routers can send requests for fragmentation with ICMP
  • It's possible to send a ping command with appropriate parameters to determine what is the ideal MTU
  • No ping with the DF bit set means that the MTU is not enough
