Spanning tree protocol

  • Connecting two switches to each other create an infinite loop:
    • no mechanism exists to block this on switches
    • easy to detect but still ruining the network
    • you're gonna have a very bad time
  • To prevent this, the spanning tree protocol (STP) helps create loop-free topologies
  • Different states to allow this:
    • blocking: avoid any loops
    • listening: listen to other ports with STP, send nothing
    • learning: add STP to the MAC address table
    • forwarding: send data
    • disabled: always disable the selected port
  • Auto config by STP to avoid any loops:
    • root port
    • designated port
    • blocked port
  • Downside of STP: some paths can be longer than necessary in the topology
  • When any change is detected, STP rebuilds the network topology and paths automatically
  • A new standard exists for STP: RSTP, which stands for Rapid STP:
    • more efficient, 6 seconds of convergence instead of 30-50 seconds
    • backwards-compatible with STP
    • very similar to STP
